行動建議

  • 🔴 立即套用 Microsoft 7 月 Patch Tuesday 更新，優先處理 SharePoint Server（CVE-2026-58644 CVSS 9.8、CVE-2026-56164 KEV 截止 2026-07-17）及 Active Directory Federation Services（CVE-2026-56155 CVSS 7.8）。SharePoint 修補後必須執行以下後修補程序：（1）依 CISA 建議排查 IIS 機器金鑰竊取指標，檢查 inetpub\wwwroot 下異常檔案及 Application Event Log 中的可疑序列化相關事件；（2）確認排除入侵指標後再輪換 IIS 機器金鑰（先輪換可能導致攻擊者在輪換過程中植入新後門）；（3）啟用 Windows Antimalware Scan Interface（AMSI）整合；（4）若 SharePoint 伺服器直接暴露於網際網路，應立即在修補完成前臨時限制存取來源 IP。
  • 🔴 立即處置 SonicWall SMA1000 雙零時差（CVE-2026-15409 CVSS 10.0、CVE-2026-15410 CVSS 7.2，CISA KEV 截止 2026-07-17）：（1）聯繫 SonicWall Support 取得熱修補程式 v12.4.3-03453 或 v12.5.0-02835 並立即套用；（2）稽核 SMA1000 存取記錄，搜尋 7 月 14 日前的異常 SSRF 相關請求或管理介面操作；（3）若發現任何入侵指標，應重新映像（硬體）或重新部署（虛擬機），並重置所有使用者及管理員密碼、TOTP 令牌；（4）確認 SMA1000 的 Appliance Work Place 介面是否直接暴露於網際網路，若非必要應限制存取來源 IP。
  • 🔴 SharePoint Server 本地部署的進階強化措施（針對本週 CISA 緊急公告）：（1）確認所有 SharePoint Server 版本均已套用 7 月 Patch Tuesday 含 CVE-2026-58644 及 CVE-2026-56164 的完整更新；（2）審查 SharePoint 過去 90 天的存取記錄，特別關注異常的匿名存取、不尋常的工作流程觸發或大量文件下載；（3）使用 Microsoft Defender 的 AMSI 偵測規則掃描 SharePoint 伺服器上的反序列化攻擊遺跡；（4）建立 SharePoint 伺服器上非預期 PowerShell 執行、新增排程工作或可疑 DLL 載入的告警機制；（5）強烈建議將 SharePoint Server 從直接網際網路暴露遷移至 Web Application Firewall 或零信任代理後方。
  • 🔴 針對 Nidec 台灣子公司 BlackField 攻擊事件，台灣製造業（尤其是外資集團的台灣子公司）應立即執行勒索軟體防禦稽核：（1）驗證所有生產系統的備份完整性，確認備份與生產環境完全隔離（含網路隔離），並測試還原程序；（2）盤點 ERP、MES、薪資及財務系統的遠端存取方式，強制要求 MFA；（3）審查台灣廠區與日本/歐美母公司間的 VPN 及遠端存取連線，確認最小權限原則；（4）稽核具有管理員或高權限帳號的第三方 IT 維護廠商存取記錄；（5）若組織使用 SonicWall 或 SharePoint，結合上述修補行動同步進行，因勒索軟體集團積極利用未修補的 VPN 及協作平台作為初始入侵點。
  • 🟡 Tenda 路由器 CVE-2026-11405 無修補程式應對措施：（1）盤點組織及員工家庭辦公環境中的 Tenda 路由器型號，確認是否屬於受影響型號（FH1201、W15E、AC10 等）；（2）立即限制路由器管理介面（通常為 192.168.0.1 或 192.168.1.1）的存取來源，確保管理介面僅可從特定有線設備存取而非 Wi-Fi 介面；（3）停用路由器遠端管理功能（WAN 側管理介面）；（4）將受影響 Tenda 路由器納入更換計畫，優先採購具持續安全韌體更新支援的品牌；（5）員工居家辦公的 Tenda 路由器若曾連接公司 VPN，應特別注意是否存在 DNS 重定向攻擊跡象（非預期網站連線失敗或證書警告）。
  • 🟡 從 AssuranceAmerica 外洩及 D1R 供應商攻擊事件提升憑證與存取管理控制：（1）強制要求所有具特權存取（VPN、雲端控制台、ERP、EDR 管理介面）的員工啟用 MFA，無例外；（2）稽核第三方供應商（IT 維護廠商、EDA 軟體供應商、自動化設備商）的存取帳號是否具備適當的最小權限設定，並定期輪換共用帳號；（3）若組織使用 Synopsys EDA 工具或 Bosch 工業設備，確認相關廠商帳號是否有存取組織環境的能力，暫時加強稽核這些帳號的存取記錄；（4）建立員工帳號異常存取的自動化偵測與告警，特別關注非工作時間登入、異常位置存取及大量資料下載行為。
  • 🟢 從拉脫維亞林業公司案例執行非核心系統修補稽核：（1）盤點組織內超過 12 個月未套用安全更新的系統，特別關注非核心業務的網頁應用程式、公司入口網站、舊版 ERP 模組及 OT/IT 整合界面；（2）確認這些系統的網路分隔狀態，若無法立即修補應確保其無法直接從外部網際網路或公司外部 IP 存取；（3）為遺留系統建立臨時監控機制，偵測異常登入或流量模式；（4）評估修補或汰換的優先順序時，應以系統對核心業務的存取路徑（而非其功能重要性）作為主要判斷依據——功能次要的系統若連接核心資料庫，其風險遠高於功能重要但完全隔離的系統。

📚 本期術語 (11)

CVE

通用漏洞揭露

公開已知漏洞的統一編號系統

KEV

已知遭利用漏洞清單

CISA 維護的已確認遭實際利用的漏洞清單

CVSS

通用漏洞評分系統

評估漏洞嚴重程度的標準化計分系統

RCE

遠端程式碼執行

攻擊者可遠端在目標系統執行任意程式碼的漏洞

SSRF

伺服器端請求偽造

利用伺服器發送請求存取內部資源的攻擊

WAF

網頁應用程式防火牆

保護網頁應用程式免受常見攻擊的安全設備

VPN

虛擬私人網路

透過加密通道在公共網路上建立私密連線

MFA

多因子驗證

要求使用者提供兩種以上驗證因子的身分驗證方式

DNS

網域名稱系統

將網域名稱轉換為 IP 位址的分散式系統

EDR

端點偵測與回應

監控端點設備以偵測和回應威脅的安全解決方案
