行動建議

  • 🔴 立即升級所有 Joomla 網站的 SP Page Builder 至 6.6.2（CVE-2026-48908，CVSS 10.0，CISA KEV，截止 2026-07-10）及 Page Builder CK 至 3.6.0（CVE-2026-56290，CVSS 10.0）。升級後執行：（1）搜尋 /images/sppagebuilder/ 及 /media/com_pagebuilderck/ 目錄中的所有 .php 檔案；（2）稽核 Joomla 後台超級管理員帳號清單，立即刪除任何電子郵件含 @secure.local 或未知來源的帳號；（3）檢查網頁伺服器存取記錄中是否有針對 uploadCustomIcon 端點的異常 POST 請求；（4）全面審查所有已安裝 Joomla 外掛的版本狀態。
  • 🔴 立即升級 Gitea Docker 至 1.26.3（CVE-2026-20896，CVSS 9.8，公告後第 13 天已確認積極利用）。升級後：（1）確認 GITEA__service__ENABLE_REVERSE_PROXY_AUTHENTICATION 設定為 false；（2）稽核程式碼倉庫存取記錄，特別關注來自 ProtonVPN 或 Tor 出口節點 IP 的異常存取；（3）稽核所有倉庫中未授權的分支、提交或 Deploy Key 修改；（4）輪換可能已洩漏的部署令牌、API 金鑰及 CI/CD 機密；（5）若組織以 Gitea 儲存 IaC 或基礎設施組態，應視為高風險優先處理。
  • 🔴 對 JADEPUFFER AI 代理勒索軟體攻擊向量執行緊急資產盤點：（1）識別組織內所有 Langflow 部署，確認已升級至 1.3.0 以上（修補 CVE-2025-3248）並套用 CVE-2026-55255 最新修補程式（CISA KEV 截止 2026-07-10）；（2）盤點所有 Nacos 組態管理伺服器，確認是否面向網際網路或僅允許內部存取；（3）稽核 AI 服務 API 金鑰（OpenAI、Anthropic、Azure AI 等）、雲端供應商憑證是否存在異常使用記錄；（4）為 AI 開發平台（Langflow、Flowise、n8n 等）建立存取異常告警機制。
  • 🔴 Accenture 外洩事件後置作業（適用所有與 Accenture 有業務往來的台灣企業）：（1）主動聯繫 Accenture 台灣確認合作專案是否受影響；（2）稽核 Azure 租用戶存取記錄，特別關注 7 月 6 日後涉及 Accenture 相關帳號的異常操作；（3）若 Accenture 曾取得組織 Azure PAT、SSH 金鑰或其他憑證，應立即輪換；（4）在 Azure Active Directory 中稽核外部來賓帳號（Guest accounts）的存取記錄。
  • 🔴 立即修補 Citrix NetScaler ADC/Gateway（CitrixBleed 2.0，CVE-2026-8451，CVSS 8.8，自 7 月 1 日起持續遭積極利用）：（1）確認版本並升級至 14.1-72.61 或 13.1-63.18 以上；（2）確認設備是否配置為 SAML Identity Provider（漏洞觸發前提）；（3）稽核 7 月 1 日後的 NetScaler 作業階段記錄，搜尋來自 146.70.139.154 或類似掃描 IP 的請求；（4）若 SAML IdP 功能未實際使用，可暫時停用作為緩解措施並立即規劃升級。
  • 🟡 針對 Oracle E-Business Suite（CVE-2026-46817，CVSS 9.8，積極利用中）：（1）確認 EBS 版本（12.2.3-12.2.15）並立即套用 Oracle 2026 年 5 月 CSP 修補程式；（2）確認 Oracle Payments File Transmission 元件是否暴露於網際網路，若是應立即限制存取；（3）稽核 6 月 27 日後的 EBS 存取記錄，搜尋針對 Payments 模組的未授權 HTTP 請求；（4）使用 EBS 處理金融支付業務的台灣金融機構及製造業，應提升稽核頻率並考慮暫時限制 Payments 模組的外部存取。
  • 🟡 防範 HalluSquatting AI 供應鏈攻擊，建立 AI 輔助程式開發的套件安全驗證流程：（1）在 CI/CD 管線加入套件完整性驗證步驟（npm audit、pip-audit、dependabot 等）；（2）建立組織核准套件清單（allowlist），AI 助理建議安裝的套件需先驗證是否存在於官方正式清單；（3）評估採用私有套件倉庫（如 Artifactory、Nexus）作為 AI 建議套件的中間審核層；（4）要求開發人員在安裝 AI 建議套件前，手動確認套件的發布者、首次發布時間及下載量等可信度指標。
  • 🟡 針對 UNK_MassTraction 大學及研究機構 Roundcube 攻擊，台灣學術機構及科研單位應執行：（1）確認 Roundcube 版本並升級（修補 CVE-2024-42009 及 CVE-2025-49113）；（2）稽核 Roundcube 伺服器上的異常 PHP 檔案，特別關注近期修改時間戳記的檔案；（3）在電子郵件閘道加強教職員及研究人員帳號的登入異常偵測；（4）半導體、國防科技、量子計算等敏感研究領域應視此為高風險威脅，考慮通報 TWCERT/CC。
  • 🟢 依 BeyondTrust CVE-2026-40138（CVSS 9.2，自主托管版本尚未修補）：若組織使用 BeyondTrust Remote Support 或 PRA 自主托管版本，立即升級至 RS 25.3.3 或 PRA 25.3.3。目前雖無確認利用活動，但 BeyondTrust 具有高知名度（CVE-2024-12356 曾被利用攻擊美國財政部），應在漏洞遭積極利用前完成預防性修補，並確認雲端托管版本已驗證 4 月修補已生效。
