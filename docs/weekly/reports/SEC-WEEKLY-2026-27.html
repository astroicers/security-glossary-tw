行動建議

  • 🔴 立即升級所有 SimpleHelp 伺服器至 5.5.16 或 6.0 RC2（CVE-2026-48558，CVSS 10.0，CISA KEV，截止 2026-07-02）。稽核 SimpleHelp 管理介面中的技術員帳號清單，刪除任何非由內部人員建立的帳號，並輪換所有技術員帳號密碼。特別注意使用 OIDC 驗證的部署——這是唯一的漏洞觸發條件。啟用 SimpleHelp 稽核記錄並回溯稽核過去 2 週的存取記錄。
  • 🔴 立即套用 Adobe ColdFusion 修補程式（APSB26-64），升級至 ColdFusion 2025 Update 10 或 ColdFusion 2023 Update 21（7 個 CVSS 10.0 漏洞，CVE-2026-48282 已確認遭利用）。優先確認 ColdFusion 伺服器是否面向網際網路，掃描 Web 根目錄是否有未預期的 .cfm/.jsp 及可執行檔，稽核 6 月 30 日後的所有檔案系統變更記錄。
  • 🔴 立即套用 Microsoft SharePoint Server 5 月修補程式（CVE-2026-45659，CVSS 8.8，CISA KEV，截止 2026-07-04，KB5002863/KB5002868/KB5002870）。即使已套用修補，必須輪換 SharePoint 伺服器的 ASP.NET machineKey——Storm-2603 的歷史 ToolShell 攻擊模式顯示攻擊者在補丁前可能已竊取金鑰以維持持久化存取。回溯稽核 5 月 2026 後的所有 SharePoint 存取記錄，確認是否有可疑的 Site Member 帳號活動。
  • 🔴 立即升級 Progress Kemp LoadMaster 至 GA v7.2.63.2 或 LTSF v7.2.54.18（CVE-2026-8037，CVSS 9.6，利用活動自 6 月 29 日起已開始）。若暫無法升級，停用或嚴格限制 LoadMaster API 存取（漏洞需 API 啟用）。稽核 6 月 29 日後的 LoadMaster 管理存取記錄，確認是否有來自未知 IP 的可疑存取。
  • 🔴 全面稽核組織 FortiGate 防火牆是否受 FortiBleed 行動影響：（1）搜尋是否有主動使用 'diagnose sniffer packet' 指令的異常診斷活動；（2）確認所有 FortiGate 管理介面是否開啟於網際網路並限縮存取；（3）重置所有 FortiGate 管理員密碼並啟用多因素驗證；（4）若組織曾使用 FortiClient EMS 7.4.5-7.4.6，立即套用修補並稽核是否有可疑可執行檔執行記錄。
  • 🔴 立即升級 Fortinet FortiClient EMS 至受影響版本以外的修補版本（CVE-2026-35616）。稽核 EMS 伺服器上是否存在名稱含 'FortiEndpoint_Patch.exe' 或類似偽裝名稱的可執行檔。搜尋端點行為記錄中是否有異常的瀏覽器憑證存取與 HTTP POST 外洩活動，特別是能源業環境。
  • 🟡 針對本週 Adobe ColdFusion、SimpleHelp、LoadMaster 三個 CVSS 10.0/9.6 漏洞的武器化時間壓縮趨勢，重新評估並優化組織的修補週期策略：（1）建立邊界設備與網際網路面向服務的緊急修補流程，目標在漏洞公告後 24 小時內完成；（2）訂閱 CISA KEV RSS 或 API 通知並設定即時告警；（3）針對高風險產品（ColdFusion、SharePoint、Fortinet、Progress 等）建立優先修補清單並追蹤版本狀態。
  • 🟡 針對 MSP 及使用 RMM 工具（SimpleHelp、ConnectWise、Kaseya 等）的組織，加強 RMM 平台的存取控制稽核：（1）盤點所有 RMM 平台的身份驗證機制，確認 SSO/OIDC 設定安全性；（2）實施 RMM 存取的條件式存取策略（IP 白名單、設備合規性）；（3）定期稽核 RMM 技術員帳號清單；（4）評估是否有受管客戶端點需要額外告知 SimpleHelp 事件潛在影響。
  • 🟡 鑒於 Supermicro 台灣伺服器異常運輸調查事件，重新稽核組織高價值伺服器硬體的採購、運輸與驗收流程：（1）確認供應商硬體固件版本完整性；（2）對關鍵基礎設施伺服器實施到貨後固件稽核；（3）評估是否引入硬體供應鏈信任驗證機制（如 NIST SP 800-193 平台固件彈性）。
  • 🟢 針對 FBI 警告的勒索軟體集團與 TeamPCP 工業化攻擊聯盟趨勢，強化組織的勒索軟體防禦縱深：（1）確認所有關鍵業務系統的離線備份策略並定期測試還原流程；（2）強化端點偵測（EDR）規則以識別 DLL 搜尋順序劫持（Storm-2603 / Warlock 攻擊手法）；（3）審查 Microsoft 365 / 本地 SharePoint 的稽核記錄保留政策，確保至少 90 天可追溯；（4）對高階主管與 IT 人員實施勒索軟體社會工程演練。
