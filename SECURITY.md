## 回報安全漏洞

如果您發現安全漏洞，請透過以下方式回報：

1. **請勿** 在公開 Issue 中回報安全漏洞
2. 請寄送電子郵件至專案維護者，或透過 GitHub Security Advisory 私下回報
3. 請提供足夠的細節讓我們能重現問題

## 回應時間

- 我們會在 **7 個工作天** 內確認收到您的回報
- 我們會在 **30 天** 內提供初步評估與處理計畫
- 修復完成後會公開致謝（除非您希望匿名）

## 範圍

本專案主要包含：
- YAML 術語定義檔案
- Python 套件程式碼
- 靜態網站產生器

以下類型的安全問題屬於本政策範圍：
- Python 程式碼中的安全漏洞
- 依賴套件的已知漏洞
- 資料驗證問題

## 致謝

感謝所有協助改善本專案安全性的貢獻者。
